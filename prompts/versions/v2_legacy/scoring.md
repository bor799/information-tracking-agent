## 输出格式

严格按以下 JSON 格式输出（不要添加任何额外文字）：

```json

  "score": <总分 0-10>,s
```

## 决策逻辑

- **总分 ≥ 7.5**: `passed: true`, `decision: "通过"`
- **6.0 ≤ 总分 < 7.5**: `passed: false`, `decision: "边缘"`
- **总分 < 6.0**: `passed: false`, `decision: "拒绝"`

## 特别过滤指令（一票否决）
- **绝不看纯代码：** 如果文章重点是展示代码实现、技术底层原理（如各类AI模型架构微调等硬核细节），且不包含任何产品落地、增长策略或如何利用其赚钱的篇幅，一律强力扣减，设置 `passed: false`。
- **绝不看老生常谈：** 如果文章只是对大众已知的概念（AI取代人类、我们要有产品思维等）的空洞重复，缺乏独特实操经验的，一并拒绝。
